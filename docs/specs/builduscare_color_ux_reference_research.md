# Buildus Care 색감 & UX 레퍼런스 정리

작성일: 2026-05-27

이 문서는 Buildus Care의 색감 방향과 UX 개선 항목을 정리한 레퍼런스 문서입니다. 코드 구현 문서가 아니라, 앞으로 화면을 정리할 때 참고할 기준입니다.

## 1. 현재 작업 상태

사용자 피드백 기준으로 현재 완료된 항목과 남은 항목을 분리했습니다.

### 완료된 항목

| 항목 | 상태 | 메모 |
| --- | --- | --- |
| 하단 global footer 사이즈 정리 | 완료 | footer는 이후 추가 수정 시 현재보다 다시 커지지 않도록 유지 필요 |
| 제품 필터 밑의 "xx개의 제품" 중복 정리 | 완료 | 제품 개수는 제품 목록 상단 기준으로 한 번만 보여주는 방향 유지 |
| 상세주소 필수화 | 완료 | 동/호수 또는 층 정보 누락 시 진행되지 않도록 처리 |

## 2. 색감 사용 방향

현재 Buildus Care에는 `크림 + 차콜 + 세이지 + 브라스 골드` 조합이 가장 잘 맞습니다.

| 역할 | 색상 | 사용 위치 |
| --- | --- | --- |
| 크림 배경 | `#f7f1e6` | 전체 페이지 배경 |
| 따뜻한 아이보리 | `#fffaf1` | 카드, 폼, sticky 패널 |
| 보조 표면 | `#eee7da` | 섹션 구분, 비활성 칩, 연한 패널 |
| 경계선 | `#d9d2c4` | 카드/폼 border |
| 차콜 | `#22211d` | 본문, 주요 CTA, 결제 버튼 |
| 흐린 본문 | `#74736a` | 설명, 보조 텍스트, helper text |
| 세이지 | `#a8b0a2` | 예약 가능, 완료 상태, 안심/관리 메시지 |
| 연한 세이지 | `#e4e8df` | 완료 단계 배경, 부드러운 상태 표시 |
| 브라스 골드 | `#b88a2b` | 가격, 추천 제품, 선택 완료 강조 |
| 골드 워시 | `#f4ead4` | 선택 상태 배경, 가벼운 강조 |

### 배경

배경은 지금처럼 크림과 아이보리 중심이 좋습니다.

- 전체 배경: `#f7f1e6`
- 카드/폼: `#fffaf1`
- 섹션 구분: `#eee7da` 또는 `#f4ead4`

순백색을 많이 쓰면 서비스가 차갑고 일반 SaaS처럼 보일 수 있습니다. 집, 욕실, 방문 서비스의 따뜻함을 유지하려면 크림 톤을 유지하는 게 좋습니다.

### 주요 버튼

주요 CTA는 차콜이 적합합니다.

- 결제하기
- 예약 확정
- 사진 견적 시작
- 주요 제출 버튼

파란색 CTA는 일반 IT 서비스 느낌이 강해져서 현재 브랜드 톤과는 덜 맞습니다.

### 세이지

세이지는 신뢰와 관리 상태를 표현하는 데 좋습니다.

- 예약 가능
- 방문 가능
- 완료된 단계
- 안심 안내
- 관리/점검 관련 메시지

다만 너무 많이 쓰면 화면 전체가 흐릿해지므로 상태 표시와 보조 배경 위주로 제한하는 게 좋습니다.

### 골드

골드는 가격과 선택 상태에만 제한적으로 쓰는 게 좋습니다.

- 가격 강조
- 추천 제품 badge
- 선택 완료
- 프로모션성 안내

골드를 메인 버튼색으로 쓰면 고급스럽기보다 광고성으로 보일 수 있습니다.

## 4. UX 개선 방향

### 4.1 결제 sticky-cta 정리

현재 남은 주요 개선 항목입니다.

권장 방향:

- 데스크탑: 우측 결제 요약 패널
- 모바일: 하단 compact CTA
- 모바일 펼침 상태: bottom sheet 형태
- 기본 상태에서는 화면을 많이 가리지 않게 구성

모바일 기본 상태에서 보여줄 정보:

- 현재 총액 또는 상담 필요 상태
- 결제 가능 시 결제 CTA
- 결제 전 상담이 더 적합하면 "카톡상담" CTA

펼쳤을 때 보여줄 정보:

- 선택 제품
- 방문 일정
- 사진 수
- 결제수단
- 약관 확인
- 결제 버튼

참고:

- [Baymard Mobile E-Commerce UX](https://baymard.com/research/mcommerce-usability)
- [Baymard Checkout Flow UX](https://baymard.com/learn/checkout-flow-ux-optimization)

### 4.2 예약 form-step-block 기능 정리

예약 단계는 사용자가 지금 어디까지 했는지 바로 이해해야 합니다.

권장 단계:

1. 사진 등록
2. 주소 입력
3. 방문 일정 선택
4. 제품 선택 / 결제

각 단계 상태:

- 완료
- 현재 진행 중
- 필수 항목 누락

완료된 단계는 접힌 요약 형태로 보여주고, 현재 단계만 충분히 넓게 열어주는 방식이 좋습니다.

참고:

- [Baymard Form Design](https://baymard.com/learn/form-design)
- [Baymard Checkout UX](https://baymard.com/learn/checkout-flow-ux-optimization)

### 4.3 제품 필터와 제품 개수 표시

이 항목은 완료되었습니다.

유지 방향:

- 제품 개수는 제품 목록 상단에 한 번만 표시
- 필터 아래에는 중복 문구를 두지 않음
- 필터 아래에는 적용된 필터 chip만 표시

예시:

- "전체 제품 24개"
- "아메리칸스탠다드 8개"
- "추천 제품 4개"

참고:

- [Baymard Product Lists UX](https://baymard.com/research/ecommerce-product-lists)
- [Baymard Filter UI](https://baymard.com/learn/ecommerce-filter-ui)

### 4.4 전체 제품 정렬과 3열 비교

이 항목은 완료되었습니다. 제품 목록은 랜덤하게 보이면 선택 피로도가 커지므로, 상단 브랜드/가격 필터와 가격 낮은 순 정렬을 기준으로 정리했습니다.

유지 방향:

- 추천 제품은 상단 카드형 유지
- 전체 제품은 데스크탑 3열 compact 카드
- 브랜드는 상단 chip 필터로 제어
- 전체 제품은 가격 낮은 순으로 비교
- 품번은 항상 표시
- 설명은 짧게 표시하고, 상세 내용은 사진 확대 모달에서 확인

브랜드별 섹션 분리는 필터와 역할이 중복되어 제외했습니다.

### 4.5 상세주소 필수화

방문 서비스에서는 상세주소가 사실상 필수입니다.

권장 문구:

- `상세주소 *`
- helper text: `동/호수 또는 층 정보를 입력해야 방문이 가능합니다.`

중요한 점:

- placeholder로만 안내하지 않기
- label에서 필수임을 명확히 보여주기
- 누락 시 바로 오류 메시지 표시

참고:

- [Baymard Required / Optional Form Fields](https://baymard.com/blog/required-optional-form-fields)

### 4.6 API 전환과 로딩 피드백

API 이동 자체를 무조건 줄이기보다, 사용자가 "멈췄다"고 느끼지 않게 만드는 것이 중요합니다.

권장 방향:

- 버튼 클릭 즉시 disabled 처리
- 버튼 문구 변경: `확인 중...`, `예약 가능 시간 불러오는 중...`
- 빈 화면 대신 skeleton 또는 기존 화면 유지
- 다음 단계에서 필요한 데이터는 가능한 미리 불러오기

우선 확인할 API:

- 서비스 목록
- 제품 catalog
- 예약 가능 slot
- 결제 준비

참고:

- [web.dev Interaction to Next Paint](https://web.dev/articles/inp)

### 4.7 전체 사이즈와 데스크탑 기준

데스크탑은 너무 넓은 1440px 기준보다 일반적인 1280-1366px에서도 보기 좋아야 합니다.

권장 방향:

- 컨텐츠 최대폭을 과하게 넓히지 않기
- 카드 높이를 줄여 한 화면 정보량 확보
- 섹션 간 여백을 줄여 스크롤 피로도 감소
- hero 영역은 너무 크게 잡지 않기

### 4.8 시공사례 카드 크기

현재 카드가 크다고 느껴질 수 있으므로 축소 방향이 맞습니다.

권장 방향:

- 이미지 높이 축소
- 카드 padding 축소
- 서비스명, 위치/유형, 결과 요약 중심
- 가격이나 작업 시간은 작게 보조 표시

### 4.9 서비스/제품 카드 크기와 3열 배치

서비스 전체 제품은 데스크탑에서 3개씩 들어가게 하는 것이 좋습니다.

권장 그리드:

- 데스크탑: 3열
- 태블릿: 2열
- 모바일: 1열

제품 카드 필수 정보:

- 브랜드
- 모델명
- 가격
- 추천/선택 상태
- 선택 CTA

제품 설명은 길게 노출하지 말고 1-2줄로 제한하는 것이 좋습니다.

참고:

- [CANCAN Furnishings](https://www.cssdesignawards.com/sites/cancan-furnishings/48235/)
- [The Furniture Gallery](https://www.cssdesignawards.com/sites/the-furniture-gallery/34062/)

### 4.10 모바일 카톡상담 CTA

모바일에서는 사용자가 빠르게 상담으로 넘어갈 수 있어야 합니다.

권장 방향:

- 모바일 sticky 또는 주요 CTA에 `카톡상담` 노출
- 결제가 부담스러운 단계에서는 `카톡상담`을 결제보다 우선
- 사진 견적 CTA와 함께 배치

예시 우선순위:

1. 카톡상담
2. 사진으로 견적받기
3. 서비스 선택

참고:

- [OneTap](https://www.onetapfix.com/)
- [FixZone](https://www.fixzone.app/)

### 4.11 모바일 대표 서비스 섹션

모바일에서는 "대표 서비스" 큰 섹션이 꼭 필요하지 않을 수 있습니다.

권장 방향:

- 제거하거나 매우 작게 축소
- 큰 카드 대신 chip 또는 간단 리스트
- 사용자가 바로 행동할 수 있는 영역 우선

대체 구조:

- "무엇을 고쳐야 하나요?"
- 서비스 chip
- 카톡상담
- 사진으로 견적받기

## 5. 페이지별 정리

### Home

- 모바일에서는 카톡상담과 사진 견적 CTA를 더 앞에 배치
- 대표 서비스는 제거 또는 축소
- 신뢰 메시지는 CTA 근처에 짧게 배치
- 배경은 크림 톤 유지

### Services

- 서비스 카드 크기 축소
- 데스크탑 3열 기준 정리
- 카드 안 정보는 서비스명, 짧은 설명, 시작 가격, CTA 중심

### Quote / Reservation

- step-block 상태 정리
- 상세주소 필수화
- sticky CTA 정리
- slot/API loading 피드백 강화

### Product Catalog

- 제품 개수 중복 표시는 완료 상태 유지
- 브랜드/가격 필터 적용
- 데스크탑 3열 compact 카드
- 모바일은 compact 카드
- 사진 확대 모달 제공

### Cases

- 카드 크기 축소
- 이미지 높이 조정
- before/after 품질이 낮으면 텍스트 요약 중심으로 정리

### Footer

- footer 사이즈 정리는 완료 상태 유지
- 모바일에서는 link group이 너무 길어지면 accordion 방식 검토

## 6. 남은 작업 우선순위
