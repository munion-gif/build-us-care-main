# 주문조회/회원 주문함 확장 메모

작성일: 2026-05-12

이번 턴에서는 정식 회원가입/로그인 시스템을 만들지 않고, 비회원 주문조회 UX를 강화했습니다.

현재 구조:

- `/orders/lookup`에서 이름 + 전화번호로 주문 목록 조회
- 조회 결과는 `/orders/[id]?accessToken=...` 기존 상세 구조로 진입
- API 응답은 주문 배열 형태라 추후 회원 주문함 UI로 재사용 가능

후속 확장 방향:

1. Magic link 기반 주문함
   - 이름 + 전화번호 확인 후 일회성 링크 발송
   - 링크에서 여러 주문을 accessToken 없이 세션 범위로 조회

2. Supabase Auth 기반 마이페이지
   - 휴대폰 인증 또는 이메일 로그인 도입
   - `customers`와 auth user 매핑 정책 필요

3. 알림 채널 연동
   - 주문조회 시 카카오/SMS로 주문함 링크 발송
   - 링크 만료와 재발급 정책 정의 필요

주의:

- 현재 accessToken은 주문 상세 진입용 안전값으로 유지합니다.
- 고객 PII는 목록 화면에서 최소 표시만 유지하고, 상세 정보는 기존 주문 상세 화면에서 확인합니다.
