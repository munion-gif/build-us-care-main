## 1. 구현 요약

이번 턴에서는 새 스키마/마이그레이션 없이 기존 주문, 예약, 사례, 관리자 운영 기능을 개선했습니다.

- `/orders/lookup`를 이름 + 전화번호 기반 주문 목록 조회로 개선
- 주문조회 결과를 단일 링크가 아닌 최신순 주문 카드 목록으로 표시
- `/cases`를 전환 중심 사례 탐색 화면으로 개선
- `/api/cases` 응답에 필터 facet, 전/후 이미지, CTA 링크, 사례 요약 정보 추가
- `/admin/orders/[id]`에 운영 수정 1차 패널 추가
- `/api/admin/orders/[id]` PATCH를 상태 변경 단일 용도에서 고객/주소/메모/예약/A/S 수정까지 확장
- 추후 회원 주문함 확장을 위한 메모 작성: `docs/order-account-roadmap.md`

## 2. 주문조회 개선 내용

### 변경 전

- 전화번호만 입력
- 최근 주문 1건의 현황 링크만 반환
- 고객 입장에서는 “내 주문내역”이라기보다 “링크 재발급” 경험에 가까웠음

### 변경 후

`/orders/lookup`에서 아래 정보를 입력합니다.

- 이름
- 전화번호

이름 + 전화번호가 일치하면 주문 목록을 최신순으로 반환합니다.

각 주문 카드 표시 항목:

- 주문번호
- 서비스명
- 주문일
- 현재 상태 문구
- 결제 여부
- 예약일/오전·오후
- 주문 상세 보기 버튼

상세 진입은 기존 구조를 유지합니다.

- `/orders/[id]?accessToken=...`

### API 변경

`POST /api/orders/lookup`

- 요청: `{ name, phone }`
- 응답: `orders[]`
- 전화번호는 숫자만 남겨 정규화
- 이름은 trim 후 비교
- rate limit 유지
- accessToken은 상세 진입 링크 생성에만 사용

조회 실패 문구:

> 일치하는 주문을 찾지 못했어요. 이름과 전화번호를 다시 확인해주세요.

## 3. 사례 페이지 UX 개선 내용

### `/cases`

사례 페이지를 “신뢰 자료 + 견적 전환 중간 단계”로 재구성했습니다.

개선 사항:

- 카테고리 탭 유지
- 서비스 유형 필터 추가
- 지역 필터 추가
- 사례 카드에 대표 완료 이미지, 서비스명, 요약, 지역/주거 태그 표시
- 카드별 CTA 추가
  - 이 서비스 견적 보기
  - 사진 판정
- 선택 사례 상세 패널 추가
  - 전/후 비교
  - 문제
  - 작업 내용
  - 공간 정보
  - 견적/사진판정 CTA

### `/api/cases`

기존 완료 사진 중심 응답을 확장했습니다.

추가 응답:

- `before_image_url`
- `summary`
- `problem`
- `work`
- `region`
- `building_type`
- `tags`
- `quote_href`
- `photo_href`
- `facets.services`
- `facets.regions`

이미지가 부족한 경우 “사진 준비 중” fallback을 표시합니다.

## 4. 관리자 수정 기능 내용

### 위치

`/admin/orders/[id]`

기존 주문 상세 상단에 `운영 수정 1차` 패널을 추가했습니다.

### 수정 가능 항목

고객/주문:

- 고객명
- 전화번호
- 주소
- 동/지역
- 아파트/단지
- 주문 메모

예약:

- 예약 날짜
- 오전/오후 슬롯

기사:

- 담당 기사 지정
- 방문 예정 시각 조정
- 기존 배정 취소 버튼 유지

A/S:

- A/S 상태
- 책임 구분
- resolved_at 처리

### 안전장치

- 저장 전 `confirm()` 확인
- 저장 성공 후 메시지 표시 및 새로고침
- 시공 시작 이후(`in_progress`, `done`, `inspected`) 예약 변경 제한
- 관리자 변경은 `events`에 `admin_order_updated`로 기록
- 새 audit 테이블은 만들지 않음

## 6. 발견한 한계

- 정식 회원가입/로그인 또는 Supabase Auth 기반 마이페이지는 이번 범위에서 제외했습니다.
- 주문조회는 이름 + 전화번호 일치 방식이므로, 동명이인/공용 전화번호 상황에서는 추가 인증이 필요할 수 있습니다.
- 관리자 수정 감사 기록은 새 audit 테이블 대신 기존 `events`에 최소 기록만 남깁니다.
- 사례 콘텐츠는 현재 완료 작업의 media 기반 자동 노출이므로, 대표 사례/숨김/정렬 CMS는 아직 없습니다.
- 프로덕션에서 예약/기사/A/S 실제 변경은 운영 영향 방지를 위해 테스트 주문의 동일 값 저장과 오류 처리 중심으로 확인했습니다.
